We are building cloud-native deployment using AWS with the help of services like IAM-EC2-ECS-ECR

I've done following steps to reach here.

  1. Created a empty GitHub.
  2. Created a EC2 server to run it as an runner.
  3. Install and start required tools inside ec2 runner, Such as git, apache, docker, httpd
  4. Gived permissions inside runner like..
      sudo usermod -aG docker ec2-user
      sudo chown -R ec2-user:ec2-user /var/www/html
  5. Attached a Iam role to EC2 for security.
  6. Create a index.html, deploy.yml
  7. Make EC2 a CI/CD Runner.
  8. Push changes to git, it will run pipeline we created.
  9. We see our content live on browser, using http://(public-ip-address).
  10. Now we insatll and start ./svc.sh to run Runner as an service. it helps in stay runner online while reboot.
  11. Now to run Dockerize CI-CD, We build dockerfile.
  12. installed and give permissions as earlier.
  13. To test locally we clone our repo to ec2-runner.
  14. Updated our pipeline with docker commands.
  15. Created health-check in between to rollback updates.
  16. Created nginx blue-green deployment.
  17. Create a ECR repo.
  18. Authenticate docker to ECR.
  19. Build and push image to ECR.
  20. Created ECS cluster
  21. Created task definition
  22. Important - Health check commmand - CMD-SHELL, curl -f http://localhost/health || exit 1
  23. Run ECS services
  24. important - Attach policy.
  25. Important - please check every typo in pipeline and AWS steps to run things smoothly.
